# Rewrite Framework

Use this framework for diagnosis and rewrite.

## Diagnosis Dimensions

| Dimension | Check |
|---|---|
| Theme | Can the video be summarized in one sentence? |
| Opening | Does the first 3-5 seconds give a reason to watch? |
| Item progression | Is it a flat list or grouped by scene/category/use? |
| Concrete details | Does each important item include visible, tactile, or usage detail? |
| Scene value | Does the viewer know when they would use the item? |
| Personal voice | Does the wording sound like the user, not a brand brief? |
| Trust | Are there small caveats, tradeoffs, comparisons, or used feedback? |
| Ending | Does it leave a ranking, takeaway, or memory point? |

## Item Rewrite Template

For each item, build 2-4 recordable sentences:

```text
我买它主要是因为 [真实原因/场景]。
最打动我的是 [具体功能/材质/版型/气味/容量]。
实际用下来 [可感知结果/小缺点/对比]。
所以如果你 [相似需求/场景]，这件会比较适合。
```

Use shorter variants for fast hauls:

```text
这件我会放在 [场景] 里用，因为 [具体原因]。
比起单纯好看，它更实用的点是 [细节]。
```

## Opening Templates

Use one of these when the original opening is too flat:

- `这期是最近真正高频用到的几件，不只是新买，也有试过之后觉得能留下来的。`
- `今天按场景来分享：通勤会用的、出门会带的、家里能提升氛围的。`
- `如果你也喜欢低调、有质感、又不太费力的东西，这期可以参考。`
- `这期里面有几件我本来只是试试，结果使用频率比想象中高。`

## Transition Templates

Use small transitions to reduce flat-list feeling:

- `前面几件偏穿搭，接下来是我觉得能提升出门效率的配饰。`
- `衣服讲完，下面这几样更偏实际生活里会一直用到的东西。`
- `如果前面是解决穿什么，后面这几件就是解决出门和家里怎么更舒服。`
- `接下来这件不是最显眼的，但使用频率可能会很高。`

## Detail Boosters

Add details from these categories:

- Action: 打开、扣上、揉一下、上身、背上、喷一下、点一根、放进包里.
- Object: 相机、防晒、薄外套、水杯、电话卡、护照、钥匙、运动小物.
- Sensory: 不黏、不闷、不扎、柔软、清爽、烟熏、木质、柑橘、垂坠.
- Result: 显比例、走久不酸、洗完不变形、喷一天还有味、早上直接穿出门.
- Tradeoff: 前几次磨脚、偏大、适合宽松、味道不适合喜欢甜香的人.

## Ending Templates

End with a concrete selection:

- `这期我最推荐的是这三件：一个是最高频，一个是最适合送人，一个是最有惊喜。`
- `如果只能留三样，我会留 [A]、[B]、[C]，分别对应通勤、旅行和家里。`
- `这期里面我自己使用频率最高的是 [item]，最适合作礼物的是 [item]。`
- `大家如果想看其中哪一件的详细上身或使用反馈，可以评论告诉我。`

## Rewrite Rules

- Keep the user's casual sentence rhythm.
- Prefer concrete life details over bigger adjectives.
- Keep product names but avoid making the script sound like a catalog.
- Preserve honest caveats when present; add caveats only when supported by the draft.
- Do not invent usage results, prices, medical effects, or unavailable performance claims.
- When details are missing, offer a fill-in placeholder rather than fabricating.
